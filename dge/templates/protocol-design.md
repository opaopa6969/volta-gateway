# DGE Template: Protocol 設計

## 概要
サービス間の通信契約（ヘッダ、JWT、API、エラー）の穴を見つける。

## 推奨キャラクター
リヴァイ (実装) + ヤン (簡素化) + 専門家

## Scene 構成

### Scene 1: 通信方向と認証
focus: 誰が誰に何を送るか、認証はどうするか
リヴァイ: 「通信方向を全部列挙しろ。漏れがあるぞ」
ヤン: 「この通信、本当に要る？」

### Scene 2: データ形式とスキーマ
focus: ヘッダ、JWT クレーム、JSON ボディの仕様
リヴァイ: 「型定義を見せろ。string じゃなくて具体的に」
ヤン: 「フィールド多すぎ。半分で足りる」

### Scene 3: エラー契約
focus: 各エラーコードの意味、リトライ可否、クライアントの対応
リヴァイ: 「401 と 403 の違いを明確にしろ」

### Scene 4: バージョニングと後方互換
focus: API バージョン、破壊的変更の扱い
ヤン: 「バージョニング要る？ 1 バージョンで十分では」
